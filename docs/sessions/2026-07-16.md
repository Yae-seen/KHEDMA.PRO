# Session log — 2026-07-16 (production outage fix)

## What happened
khedma.pro was returning `404: NOT_FOUND` on every route despite the domain being
live. Diagnosed and fixed three stacked problems (site now serves 200 everywhere).

## Root causes & fixes (in order found)
1. **Vercel build failing** — `The Edge Function "middleware" is referencing
   unsupported modules: @/.` The Supabase session-refresh `middleware.ts` used the
   `@/` path alias, which Vercel's Edge bundler rejects (local `next build` was
   lenient). → Removed `middleware.ts` (`c33d563`). Re-add Edge-safe when Supabase
   is provisioned.
2. **Production branch mismatch** — Vercel's Production Branch was `main`, the repo
   is `master`. Every push deployed as a *Preview*, never promoted to Production. →
   Set Production Branch to `master` (Settings → Environments → Production).
3. **Framework Preset = `null`** (the real killer) — `get_project` returned
   `"framework": null`, so Vercel built the Next.js app but never wired its routing/
   output; the deployment's OWN `.vercel.app` URL 404'd every route. → Added
   `vercel.json` `{"framework":"nextjs"}` (`ba259e5`).

## Diagnosis method (reusable)
- DNS lookup proved the domain pointed at Vercel correctly (ruled out DNS).
- Vercel MCP `get_project` (framework:null) + `web_fetch_vercel_url` on the
  deployment's own URL (404) proved it was NOT a domain issue but the deployment
  serving nothing → framework preset.

## Analytics activated (same day)
Owner supplied GA4 `G-NQFV0NQGXH` and a Google tag `GT-MR5XFNM9`.

- **`GT-MR5XFNM9` is a Google tag (`gtag.js`), NOT a GTM container.** GTM containers
  are `GTM-XXXXXXX` and load via `gtm.js` + an `ns.html` `<noscript>` iframe; a `GT-`
  id put in that path would 404. The `GT-` tag routes to the same GA4 property as the
  `G-` id, so installing the `G-` measurement id alone captures all data, no double count.
- Committed the GA4 id as a **default** in `components/site-analytics.tsx` and
  `next.config.ts` (public, non-secret id — ships in page HTML regardless). This makes
  analytics live on deploy with no manual Vercel env step; env vars still override.
- CSP gotcha handled: `next.config.ts` only whitelists Google domains when a GA/GTM id
  is present, so the default had to be added in BOTH files or the tag would load but be
  CSP-blocked. Local `next build` green.
- **Owner action:** verify GA4 → Reports → Realtime shows traffic ~30 min post-deploy.
